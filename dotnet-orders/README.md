# .NET Orders Service

An ASP.NET Core 8 minimal-API service with a multi-step cart checkout, built to
showcase Speedscale proxymock's `smart_replace_recorded` transform. The flow
hands you a `cartId`, an `itemId`, and an `orderId`, and each one has to be copied
from a response into a later request by hand. proxymock detects every one of
those correlations and recommends a transform that does the substitution for you
on replay ‚Äî no more pasting ids.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/healthz` | Health check |
| `POST` | `/carts` | Start a cart and add an item; returns a new `cartId` + `itemId` |
| `POST` | `/carts/items/update` | Update item quantity (`cartId` + `itemId` in the body) |
| `POST` | `/carts/checkout` | Checkout a cart (`cartId` in the body); returns a new `orderId` |
| `POST` | `/orders/confirm` | Confirm / pay an order (`orderId` in the body) |
| `GET` | `/carts` | List carts |
| `GET` | `/orders` | List orders |

### The correlated ids

| id | generated by | reused in |
|----|--------------|-----------|
| `cartId` | `POST /carts` response | `update` and `checkout` request bodies |
| `itemId` | `POST /carts` response | `update` request body |
| `orderId` | `POST /carts/checkout` response | `orders/confirm` request body |

Each id is returned in a response **body** and reused in a later request **body**
(not the URL path) ‚Äî that's what proxymock's token-based correlation keys off.

## Quick Start

### Docker Compose

```bash
docker compose up -d
```

Service is available at `http://localhost:8082`.

### Run Locally

Requires .NET 8 SDK. No Docker needed ‚Äî the app listens on `:8082` by default
(set via `Urls` in `appsettings.json`), so a plain `dotnet run` matches the
scripts.

```bash
dotnet run        # or: make local  ‚Äî both listen on :8082
```

### Make Targets

```bash
make help          # list all targets
make build         # dotnet build
make local         # run locally on port 8082 (plain dotnet run)
make record        # record the order flow with proxymock (launches the app too)
make up            # docker compose up -d
make down          # docker compose down
make docker-build  # build Docker image
make health        # curl /healthz
make kube          # kubectl apply k8s/base/
make kube-clean    # kubectl delete k8s/base/
```

## Demo Flow

The flow lives in [`test.http`](./test.http) ‚Äî open it in VS Code (REST Client),
Visual Studio, or Rider and run each request with the "Send Request" button, top
to bottom.

1. **Start a cart + add an item** ‚Äî the response returns a `cartId` and an `itemId`.
2. **Update the quantity** ‚Äî paste **both** the `cartId` and the `itemId` from
   step 1 into the body (two pastes).
3. **Checkout** ‚Äî paste the `cartId` again; the response returns a new `orderId`.
4. **Confirm the order** ‚Äî paste the `orderId` from step 3.

Four steps, three different ids, copied by hand ‚Äî and one transposed character
means a `404`. That hand-editing is exactly what proxymock's Smart Replace
removes: record the flow once and it captures each live id (`cartId`, `itemId`,
`orderId`) and substitutes it on replay, so you never paste an id again.

### Record it with proxymock

`make record` launches the app *and* the recorder in one window. Point
`test.http`'s `@host` at the recorder (`http://localhost:4143`) and run the
requests, then open `proxymock web`:

```bash
make record          # window 1: app + recorder (Ctrl-C to stop)
# in test.http, set @host = http://localhost:4143, then Send each request
proxymock web        # see the "Correlated ID" smart_replace_recorded recommendation
```

## Storage

Carts and orders are kept in memory only. The store is reset when the process restarts ‚Äî no database required.

## Configuration

| Env var | Default | Description |
|---------|---------|-------------|
| `ASPNETCORE_URLS` | `http://+:8082` | Listen address |
| `ASPNETCORE_ENVIRONMENT` | `Production` | Runtime environment |
